# get-content

Перетворює посилання на чистий текст, готовий для аналізу і написання нотаток.

## YouTube

**Мета:** отримати транскрипцію відео.

### Основний спосіб — YouTubeTranscript.com

1. Витягти VIDEO_ID з URL:
   - `https://www.youtube.com/watch?v=VIDEO_ID` → VIDEO_ID
   - `https://youtu.be/VIDEO_ID` → VIDEO_ID

2. Завантажити через WebFetch:
   ```
   URL: https://youtubetranscript.com/?v=VIDEO_ID
   ```

3. Знайти текст транскрипції на сторінці (секція з субтитрами).

### Fallback — Playwright якщо WebFetch не дав тексту

1. `mcp__playwright__browser_navigate` → `https://www.youtube.com/watch?v=VIDEO_ID`
2. Натиснути `...` (More) під відео
3. Натиснути "Show transcript"
4. `mcp__playwright__browser_snapshot` або `get_page_text` — скопіювати текст транскрипції

### Формат виводу
```
[Транскрипція відео]
Назва: ...
Автор каналу: ...

<текст субтитрів блоками по абзацах>
```

## Стаття / блог

**Мета:** отримати чистий текст без навігації, реклами, коментарів.

### Основний спосіб — WebFetch

```
Інструмент: WebFetch
URL: <посилання на статтю>
```

WebFetch автоматично повертає очищений текст сторінки.

### Fallback — Playwright якщо сторінка JS-рендерована

1. `mcp__playwright__browser_navigate` → URL статті
2. `mcp__Claude_in_Chrome__get_page_text` або `mcp__playwright__browser_snapshot`
3. З отриманого тексту прибрати навігацію/меню/футер вручну

### Формат виводу
```
[Текст статті]
Джерело: ...
Автор: ...

<основний текст статті>
```

---

## Після отримання тексту

Використати отриманий текст для:
1. Аналізу і виділення ключових тез про книгу
2. Заповнення поля `source_author_notes` у файлі `books/[slug].md`
3. Формулювання власних думок для `my_notes`

---

## Edge cases

| Проблема | Рішення |
|----------|---------|
| YouTube без субтитрів | Повідомити юзера, запропонувати auto-generated |
| Сайт блокує ботів (403) | Спробувати Playwright замість WebFetch |
| Дуже довгий текст (> 50k знаків) | Попросити юзера вказати конкретні розділи |
| Пейвол / авторизація | Повідомити юзера |
