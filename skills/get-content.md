# get-content

Перетворює посилання на чистий текст, готовий для аналізу і написання нотаток.

## YouTube

**Інструмент:** скіл `youtube-transcribe` (Supadata API).

```
Skill: youtube-transcribe
Args: <URL відео>
```

Скіл повертає очищений текст транскрипції. Далі одразу аналізувати.

### Формат виводу після транскрипції
```
Назва відео: ...
Канал: ...

<чистий текст транскрипції>
```

---

## Стаття / блог

**Інструмент:** `WebFetch`

```
URL: <посилання на статтю>
Prompt: отримати чистий текст статті без навігації, реклами, коментарів
```

### Fallback — якщо WebFetch не дав тексту (JS-рендеринг)

1. `mcp__playwright__browser_navigate` → URL
2. `mcp__playwright__browser_snapshot` → витягти основний текст

---

## Після отримання тексту → книги

1. Виділити всі згадані книги / серії
2. Для кожної — сформувати `books/[slug].md` з полями:
   - `source_author_notes` — тези, оцінки, контекст з відео/статті
   - `my_notes` — залишити порожнім якщо немає власних думок
3. `git add . && git commit -m "add: назви книг" && git push`

---

## Edge cases

| Проблема | Рішення |
|----------|---------|
| Відео без субтитрів | Supadata поверне помилку — попросити юзера надати текст вручну |
| Стаття за пейволом | Повідомити юзера |
| Дуже довгий текст (> 50k) | Попросити уточнити які саме книги цікавлять |
| Відео — огляд багатьох книг | Створити файл для кожної книги окремо |
